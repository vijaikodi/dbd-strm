# Debrid-Link STRM Generator for Jellyfin/Plex

This project generates Jellyfin‑compatible `.strm` files and basic NFO metadata from your Debrid‑Link seedbox using the Debrid‑Link API. 
The `.strm` files contain HTTPS URLs that point directly to your Debrid‑Link media, so Jellyfin can stream content without needing local copies. [github](https://github.com/orgs/jellyfin/discussions/11448)

## Features

- Connects to Debrid‑Link via API (`DBD_API_KEY`, `DBD_API_URL`, `DBD_SDBX_ID`).  
- Recursively scans your seedbox folder and filters video/audio files above a configurable size threshold.  
- Uses `guessit` to parse filenames and builds Jellyfin‑friendly folder structures and names:
  - Movies: `movies/Title (Year)/Title (Year).strm`
  - TV shows: `tvshows/Show/Season 01/Show - S01E01.strm`  
- Generates minimal NFO files (movies and episodes) alongside `.strm` files to help Jellyfin with metadata scraping.  
- Cleans up obsolete `.strm` files that no longer correspond to files in your Debrid‑Link seedbox.  
- Can run once or in a loop, re‑scanning every **N minutes** inside a Docker container (via `DBD_SCAN_INTERVAL_MIN`). [digitalocean](https://www.digitalocean.com/community/tutorials/python-time-sleep)

## How it works with Jellyfin

Jellyfin treats `.strm` files as pointers to external URLs. [github](https://github.com/jellyfin/jellyfin/issues/13974)
For each Debrid‑Link file, this tool writes a `.strm` file whose contents is the Debrid‑Link HTTPS URL; when you play it, the client directly fetches from Debrid‑Link (or via Jellyfin if transcoding is needed). [github](https://github.com/jellyfin/jellyfin/issues/16289)

You create separate Jellyfin libraries for:

- Movies → `.../movies`  
- TV Shows → `.../tvshows`  
- (Optional) Music → `.../music`

and point them to the output folder generated by this tool.  

## Environment variables

The container/script is configured entirely with environment variables (no arguments needed beyond the target directory). [collabnix](https://collabnix.com/how-to-set-environment-variables-in-docker-compose/)

Required:

- `DBD_API_KEY` – Debrid‑Link API token for your account.  
- `DBD_API_URL` – Debrid‑Link API base URL (e.g. their v2 REST root). [debrid-link](https://debrid-link.com/api/v2)
- `DBD_SDBX_ID` – Debrid‑Link folder/seedbox ID to scan (root of the media tree you want to expose).  

Optional:

- `DBD_SCAN_INTERVAL_MIN` – if `> 0`, the script will run, then sleep this many minutes and run again in a loop; `0` or unset → run once and exit. [digitalocean](https://www.digitalocean.com/community/tutorials/python-time-sleep)

## Running with plain Python

### Requirements

- Python 3.10+  
- `pip install -r requirements.txt` with:

```txt
httpx
python-dotenv
guessit
```

### Steps

1. Clone the repo and install dependencies:

   ```bash
   git clone https://github.com/youruser/debridlink-strm-generator.git
   cd debridlink-strm-generator
   pip install -r requirements.txt
   ```

2. Create a `.env` file in the project root:

   ```ini
   DBD_API_KEY=your_debridlink_api_token
   DBD_API_URL=https://debrid-link.com/api/v2
   DBD_SDBX_ID=seedbox
   DBD_SCAN_INTERVAL_MIN=0
   ```

3. Run the script, pointing to your output directory:

   ```bash
   python dbd-strm.py /path/to/output
   ```

This will create/update `.strm` and NFO files under the output folder and clean up obsolete `.strm` files.  

## Docker usage

The Docker image wraps the script and expects a volume mounted at `/data`, where it writes `.strm` and NFO files.  

Example `docker run`:

```bash
docker run --rm \
  -v /srv/debrid-strm:/data \
  -e DBD_API_KEY=your_api_key \
  -e DBD_API_URL=https://api.debrid-link.com/v2 \
  -e DBD_SDBX_ID=your_seedbox_folder_id \
  -e DBD_SCAN_INTERVAL_MIN=30 \
  yourdockeruser/dbd-strm:latest
```

- `/srv/debrid-strm` on the host will contain `movies/`, `tvshows/`, `music/`.  
- `DBD_SCAN_INTERVAL_MIN=30` makes the container rescan every 30 minutes; set to `0` to run once and exit. [spacelift](https://spacelift.io/blog/docker-run-environment-variables)

## Docker Compose example

You can manage the container with Docker Compose and a `.env` file for secrets, which is a common best practice for Docker stacks. [geshan.com](https://geshan.com.np/blog/2023/08/docker-compose-environment-variables/)

Create `.env` (next to `docker-compose.yml`):

```env
DBD_API_KEY=your_api_key
DBD_API_URL=https://api.debrid-link.com/v2
DBD_SDBX_ID=your_seedbox_folder_id
DBD_SCAN_INTERVAL_MIN=30
OUTPUT_PATH=/srv/debrid-strm
IMAGE_TAG=latest
```

Then create `docker-compose.yml`:

```yaml
version: "3.8"

services:
  dbd-strm:
    image: yourdockeruser/dbd-strm:${IMAGE_TAG}
    container_name: dbd-strm
    restart: unless-stopped
    environment:
      DBD_API_KEY: "${DBD_API_KEY}"
      DBD_API_URL: "${DBD_API_URL}"
      DBD_SDBX_ID: "${DBD_SDBX_ID}"
      DBD_SCAN_INTERVAL_MIN: "${DBD_SCAN_INTERVAL_MIN}"
    volumes:
      - /srv/debrid-strm:/data
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro

    # optional: limit resources, set TZ, etc.
    # environment:
    #   TZ: "Europe/Paris"
```

Run:

```bash
docker compose up -d
```

Compose will inject the environment variables from `.env` into the service and mount your output directory on the host to `/data` in the container. [apxml](https://apxml.com/courses/docker-for-ml-projects/chapter-6-docker-compose-ml-workflows/compose-environment-variables)

## Pointing Jellyfin to the generated libraries

On your Jellyfin server, add new libraries:

- **Movies**  
  - Content type: Movies  
  - Folder: `/srv/debrid-strm/movies`  

- **TV Shows**  
  - Content type: TV Shows  
  - Folder: `/srv/debrid-strm/tvshows`  

- **(Optional) Music**  
  - Content type: Music  
  - Folder: `/srv/debrid-strm/music`  

Trigger a library scan; Jellyfin will index `.strm` files and use your NFOs and naming to fetch metadata. [reddit](https://www.reddit.com/r/jellyfin/comments/nzs4fj/how_do_i_create_jellyfin_playable_strm_files_from/)
